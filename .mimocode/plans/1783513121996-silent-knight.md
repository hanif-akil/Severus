# Kotlin Migration Plan ‚Äî Severus (Simple Keyboard)

## Overview

Convert all 91 Java source files to Kotlin. No AndroidX migration, no build script DSL change ‚Äî just language migration.

## Build Configuration Changes

### 1. Root `build.gradle` ‚Äî Add Kotlin Gradle Plugin

```groovy
buildscript {
    repositories {
        mavenCentral()
        google()
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:9.2.1'
        classpath 'org.jetbrains.kotlin:kotlin-gradle-plugin:2.1.20'
    }
}
```

### 2. `app/build.gradle` ‚Äî Apply Kotlin Plugin

```groovy
apply plugin: 'com.android.application'
apply plugin: 'kotlin-android'

android {
    // ... existing config unchanged
}

dependencies {
}
```

### 3. `gradle.properties` ‚Äî Enable Built-in Kotlin

Change `android.builtInKotlin=false` ‚Üí `android.builtInKotlin=true` (if AGP 9.2.1 supports it; otherwise the explicit plugin above handles it).

## Source Conversion

### Strategy: In-Place Conversion

1. Create `app/src/main/kotlin/rkr/simplekeyboard/inputmethod/` directory tree
2. Convert each `.java` file to `.kt` in the matching package under `kotlin/`
3. Delete all `.java` files from `app/src/main/java/`
4. Remove the now-empty `java/` directory

### Kotlin Conversion Patterns

| Java Pattern | Kotlin Equivalent |
|---|---|
| Eager singletons (`static final sInstance`) | `object` declarations |
| Static methods (`public static foo()`) | Top-level functions or `companion object` |
| `new Handler() { ... }` anonymous classes | SAM conversion or lambda |
| `public final` fields | `val` properties |
| `public` mutable fields | `var` with backing field if needed |
| `LeakGuardHandlerWrapper<T>` | Same class, Kotlin generics syntax |
| `synchronized` blocks | `@Synchronized` or `synchronized(lock) { }` |
| Null checks / `@Nullable` | Kotlin null types (`?`) |
| `instanceof` / casts | `is` / `as` |
| Builder patterns | Kotlin DSL or named parameters |
| `final` classes | `final` (same; Kotlin classes are final by default) |

### Special Considerations

1. **`Key.java` (906 lines)** ‚Äî Largest single file. Has 3 static inner classes. Convert inner classes to nested Kotlin classes.
2. **`LatinIME.java` (954 lines)** ‚Äî Main IME service with `UIHandler` static inner class, anonymous `BroadcastReceiver`. Convert handler to `companion object` + inner class.
3. **`Settings.java`** ‚Äî Eager singleton with `ReentrantLock`. Use `object` + `@JvmStatic` if needed for Java interop (none expected since all Java is being converted).
4. **No AndroidX** ‚Äî The deprecated `android.preference.*` APIs are used directly. Keep as-is (not in scope).
5. **No tests** ‚Äî Nothing to update on the test side.
6. **ProGuard** ‚Äî Keep the existing `-keep` rules; they reference class names which won't change.

## Post-Conversion Cleanup

1. Delete empty `app/src/main/java/` directory tree
2. Verify `app/src/main/kotlin/` contains all converted files
3. Run `./gradlew assembleDebug` to confirm compilation
4. If ProGuard issues arise, add Kotlin-specific keep rules

## Verification

1. `./gradlew assembleDebug` ‚Äî must compile without errors
2. `./gradlew assembleRelease` ‚Äî must compile and ProGuard successfully
3. Manual APK inspection ‚Äî confirm the app installs and the keyboard functions

## Risk Assessment

- **Low risk**: No external dependencies to conflict with Kotlin stdlib
- **Low risk**: No AndroidX migration mixed in
- **Medium risk**: Deprecated `PreferenceActivity`/`PreferenceFragment` usage may generate Kotlin compiler warnings (not errors)
- **Medium risk**: `Key.java` and `LatinIME.java` are complex ‚Äî careful conversion needed
